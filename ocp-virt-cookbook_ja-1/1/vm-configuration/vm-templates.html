仮想マシンテンプレートの作成

概要

ゴールデンイメージの概念に似ています。これらは、新規の VirtualMachines のデータソースとして使用される特定のディスクイメージを準備します。テンプレートは、これをさらに進め、 VirtualMachine マニフェストをテンプレート化します。このマニフェストは、仮想ハードウェアとその他のものを指定します。

  • インスタンスタイプ（vCPU コア数と RAM サイズを推論）

  • マシンの好み（CPU トポロジーとファームウェアタイプ）

  • ネットワークとインターフェース

  • ディスクとボリューム

  • 事前定義されたラベルとアノテーション

これにより、OpenShift Virtualization コンソールからもコマンドラインからも、マシン仕様の完全なデプロイメントが高速で再現可能になります。

OpenShift Virtualization コンソールから簡単に VirtualMachine を作成できます。「テンプレートから」オプションを選択し、例えば rhel9-server-large、fedora-desktop-medium などのテンプレートを選択します。

簡単に言えば、このガイドでは、仮想マシンテンプレートを選択、デプロイ、およびカスタマイズする方法について説明します。

バージョン:

OpenShift 4.21

前提条件

このガイドを完了するためには、以下のものが必要です。

  • バッシュまたは互換性のあるシェル環境（zsh はテストされました）。

  • OpenShift クライアントツール oc。

  • OpenShift クラスターに OpenShift Virtualization がインストールされている。

  • テキスト（またはコード）エディターや IDE アプリケーション。

ここでは、テンプレートパラメーターの名前、フィールド値、説明がリストされます。

各パラメーターを以下に説明します。

  • NAME - この場所では、テンプレートから作成された仮想マシンの一意の名前生成が行われます。 [a-z0-9] の文字クラス（シェルまたは正規表現スタイル）とカウント ({16}) を注意してください。これらは VM 名の後半を埋めるために使用されます。

  • DATA_SOURCE_NAME - テンプレートからクローンする DataSource（またはゴールデンイメージ）の名前。

  • DATA_SOURCE_NAMESPACE - DataSource が存在する名前空間です。 デフォルトのイメージは、openshift-virtualization-os-images プロジェクトにあります。

  • CLOUD_USER_PASSWORD - デフォルトの cloud-user ユーザーのパスワードが生成されます。 NAME パラメーターと同様に、[a-z0-9] の文字クラスとカウント ({4} 3 つのダッシュ区切りのグループ) を注意してください。

  • テンプレート内のパラメーターは、テンプレートされたマニフェストでパラメーターの名前を参照することで使用されます。 これは、バッシュシェルの変数名と参照（それぞれ）のスタイルと構文に似ています。

ローカルに rhel9-server.template.yaml ファイルを保存したら、テンプレートをテストし、 oc process コマンドを使用してレンダリングされた状態を確認できます。

oc process -f rhel9-server.template.yaml -o yaml

レンダリングされたテンプレートの YAML 出力が表示されるはずです。 生成された出力は、VirtualMachine テンプレートをリスト項として含む List オブジェクトです。

  • OpenShift テンプレートでは、1 つの Kubernetes オブジェクトがテンプレート内に表現されることがあります。

  • OpenShift Virtualization では、単一の VirtualMachine オブジェクトでテンプレートを作成できます。

ステップ 2: テンプレートから仮想マシンのデプロイ

このセクションでは、CLI を使用してテンプレートから VirtualMachine をデプロイする方法を説明します。 これは、oc process コマンドの出力を oc create（または oc apply）にパイプすることで実現できます。

まず、template-example プロジェクト（名前空間）を作成して、RHEL 9 VirtualMachine を格納します。

oc new-project template-example

次に、rhel9-server.template.yaml ファイルを処理し、出力を oc apply にパイプします（oc create も機能します。これは新しい VirtualMachine オブジェクトです）。

oc process -f rhel9-server.template.yaml | oc create -n template-example -f -

現在の名前空間で作成されたリソースを一度確認します。

oc get all -n template-example

DataVolume は、PendingPopulation になり、VirtualMachine は Stopped 状態になります。

新しい rhel9-server VM の名前を含むシェル変数を作成します。

export RHEL9_SERVER_VM=$(oc get vm -o custom-columns=:metadata.name --no-headers -n template-example)

変数を確認して、VM 名が正しいこと、および変数が空でないことを確認します。rhel9-abcdef0987654321 のような値が表示されるはずです。

echo $RHEL9_SERVER_VM

VM 名を virtctl で開始コマンドを実行できます（以下に VM 名を置き換えます）。

virtctl start $RHEL9_SERVER_VM

oc get vm を数回（または -w フラグを使用して 1 回）実行して、VM の実行状態が Stopped から Provisioning へと Running に変化することを観察します。

コンソールへのアクセスについては、ステップ 4 に進みます。これは CLI からのコンソールアクセスをカバーします。

ステップ 3: テンプレートのカスタマイズ

このセクションでは、カスタマイズするために rhel9 テンプレートのコピーを作成し、いくつかの変更をカスタムテンプレートに適用します。

前回作成したプロジェクト/名前空間（ステップ 2 で作成）に続きます。

oc project template-example

rhel9-server.template.yaml ファイルを rhel9-custom.template.yaml という名前のコピーを作成します。

cp rhel9-{server,custom}.template.yaml

重要: カスタムテンプレートを最終的に作成する前に、rhel9-custom.template.yaml ファイルに提供された YAML データをペーストする必要があります。

カスタムテンプレートがレンダリングされることを確認するために、oc process を使用します。

oc process -f rhel9-custom.template.yaml -o yaml

カスタムテンプレートが正しくレンダリングされるので、ステップ 4 に進みます。

ステップ 4: 最終テストとクリーンアップ

このセクションでは、ステップ 2 のようにカスタムテンプレートをデプロイし、このチュートリアルで作成されたリソースをクリーンアップします。

rhel9-custom テンプレートとして新しい VirtualMachine を起動します。

oc process -f rhel9-custom.template.yaml | oc apply -n template-example -f -

ソースボリュームのクローンが完了した後、rhel9-custom VM は実行状態になります。 oc get vm を確認します。

oc get vm -n template-example

rhel9-custom VM の名前を含むシェル変数を作成します。

export RHEL9_CUSTOM_VM=$(oc get vm -o custom-columns=:metadata.name --no-headers -n template-example | grep custom)

注: 元の rhel9 VM に接続したい場合は、$RHEL9_CUSTOM_VM を RHEL9_SERVER_VM に置き換えます。

パスワードを表示するために、VM の YAML 出力を表示し、記録またはコピーします。

oc describe vm $RHEL9_CUSTOM_VM | grep password

以前に取得したパスワードを使用して、virtctl を使用してコンソールにログインします。

virtctl console $RHEL9_CUSTOM_VM

ENTER を押してログインプロンプトを表示します。 CTRL+] を押してコンソールビューを終了します。

注: クラスター全体にテンプレートを OpenShift Virtualization コンソール
