OpenShift 仮想化クックブック

紹介

OpenShift 仮想化でワークロードを開始およびデプロイするための実践的なフィールドガイド。

このクックブックは、YAML、CLI ワークフロー（必要ではありませんが強く推奨されます）、Kubernetes/OpenShift の基本的な概念についての理解が必要です。理論的な学習を補完し、置き換えるわけではありません。実世界の実装を促進します。

(最終更新日: {build-date})

前提条件

このクックブックを効果的に使用するには、以下のものが必要です。

  • クラスター: OpenShift 4.18+ に OpenShift 仮想化オペレーターがインストールされている。

  • ツール: oc, kubectl, および virtctl。（オプション: kustomize, helm, ansible）。

  • 知識: YAML についての熟練度、CLI ワークフロー（必須ではありませんが強く推奨されます）。Kubernetes/OpenShift の概念についての基本的な理解。

すべての 開始 モジュールを確認し、特に新しい技術に触れている場合は、環境を準備または検証してください。

クックブック モジュール

左メニューから 1 つのモジュールを選択して始めてください。

  • 開始 - 必須のツールのセットアップと環境検証。

  • ストレージ構成 - 仮想マシンのストレージ設定。

  • ネットワーク構成 - ネットワークおよび IP 設定ガイド。

  • 仮想マシン構成 - VM アクセス、DNS、ノード配置、テンプレート。

  • VM パフォーマンスチューニング - CPU ピニング、NUMA、hugepages、リアルタイム VM。

  • VM ライフサイクル管理 - ゴールデンイメージ、クローン、スナップショット、ブート構成。

  • YAML マニフェスト ライブラリ - デプロイ用の準備ができたマニフェストの例のライブラリー。

  • API 統合 - クローン、エクスポート、マシントポロジーなどの API コンポーネントのアーキテクチャと機能。
