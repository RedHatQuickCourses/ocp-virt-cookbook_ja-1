API統合

概要

このモジュールでは、OpenShift VirtualizationのAPIについて、コンポーネントAPIオブジェクトの概要と各オブジェクト内の主要フィールドを含めて説明します。

APIは強力な機能であり、OpenShift Virtualizationと基盤となるOpenShift Container Platformの両方とのより緊密な製品統合に活用できます。

内容

このセクションで扱うトピックは次のとおりです。

  • 仮想化に関連する主要なAPIオブジェクト（仮想マシン、ネットワーク、ストレージなどのドメイン別に分類）

  • 各APIオブジェクトの役割

  • APIオブジェクト間の関係（一部は他のオブジェクトのより高レベルな抽象化です）

  • 各APIオブジェクトの必須フィールド（および各フィールドの役割と許容値）

  • より高レベルなAPI機能（スナップショット、リストア、クローニング）

  • オブジェクトの状態（VM、スナップショット、クローニング）を監視および管理する方法

  • 他のクラスターにインポートするためにVMとボリュームをエクスポートする方法

作業を開始する前に

このセクションを完了するには、以下の前提条件が必要です。

  • OpenShift VirtualizationがインストールされたOpenShift v4.18+クラスター

  • クラスターレベルの管理者アクセス（またはテスト用またはシングルテナントクラスター用の`kubeadmin`アカウント資格情報）

  • REST APIと関連するHTTP CRUD操作（GET、PUT、POST、PATCH、DELETE）、トークンベース認証、jsonなどの理解

  • REST API開発ツール（Postman、cURLなど）

次のステップ

このモジュールを完了すると、OpenShift Virtualizationと独自の製品を統合し始めるのに十分な理解が得られるはずです（願わくば）。 このガイドでは、将来の（そして現在の）パートナーの市場投入までの時間を短縮するために、他のベンダーとの統合で遭遇した注意点や問題点を挙げるように努めました。
