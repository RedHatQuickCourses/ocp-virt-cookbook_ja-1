OpenShift仮想化オペレーターのインストールと構成

概要

このチュートリアルでは、OpenShift仮想化オペレーターのインストールと構成方法を紹介します。このオペレーターを使用すると、OpenShiftクラスター上で仮想マシンとコンテナワークロードを同時に実行できます。OpenShift仮想化は、upstreamのKubeVirtプロジェクトに基づいており、エンタープライズレベルの仮想化機能を提供します。

  • Webコンソールを使用したインストール

  • [コマンドラインを使用したインストール]

前提条件

  • OpenShift 4.18+ がインストールされ、サポートされているプラットフォームで実行中

  • クラスター管理者アクセス権限: オペレーターのインストールとクラスターリソースの構成に必要

テスト済みバージョン:

OpenShift 4.18, 4.19, 4.20, 4.21

ハードウェアおよびソフトウェア要件

OpenShift仮想化をインストールする前に、クラスターがハードウェア、オペレーティングシステム、ストレージ、ネットワーク要件を満たしていることを確認してください。

詳細な要件、CPU仕様、サポートされるプラットフォーム、ストレージに関する考慮事項、リソースオーバーヘッドについては、クラスターをOpenShift仮想化に準備するを参照してください。

ヒント: ストレージ構成については、オペレーターインストール後にデフォルトストレージクラスの構成を参照してください。

Webコンソールを使用したインストール

Webコンソールは、OpenShift仮想化のガイド付きインストールを提供します。

ステップ1: OperatorHubにアクセス

  • クラスター管理者権限を持つユーザーとしてOpenShiftWebコンソールにログインします

  • Operators → OperatorHubに移動します

  • キーワードフィルターフィールドに、`Virtualization`を入力します

ステップ2: オペレーターのインストール

  • Red Hatソースラベルを持つOpenShift Virtualizationタイルを選択します

  • オペレーターの情報を確認し、`インストール`をクリックします

  • オペレーターのインストールページで、デフォルト設定を維持し、`インストール`をクリックします

重要: 手動承認が利用可能ですが、不適切なバージョンの組み合わせを実行するリスクがあるため、推奨されません。手動を選択する場合は、その意味を完全に理解していることを確認してください。

  • オペレーターのインストールが完了するまで待ちます

ステップ3: HyperConvergedリソースの作成

オペレーターのインストールが完了したら、`HyperConverged`を作成します

+ image::install-6-create-hyperconverged.png[オペレーターインストール後の`HyperConverged`の作成ボタン]

  • デフォルト設定を維持し、ページの下部にスクロールします

  • `作成`をクリックしてOpenShift仮想化をデプロイします

ステップ4: インストールの確認

すべての条件が成功を示すまで待ちます。HyperConvergedリソースのステータスは、デプロイメントの進捗を示します。

インストールが完了すると、オペレーターは`Operators`リストで`インストール済み`として表示されます。

コマンドラインを使用したインストール

Webコンソールにアクセスできない自動化デプロイメントの環境や、自動化されたデプロイメントの環境では、CLIインストール方法を使用します。

ステップ1: Namespace、OperatorGroup、およびSubscriptionの作成

すべての必要なリソースを含む1つのマニフェストファイルを作成します:

oc apply -f - <<EOF
apiVersion: v1
kind: Namespace
metadata:
  name: openshift-cnv
---
apiVersion: operators.coreos.com/v1
kind: OperatorGroup
metadata:
  name: kubevirt-hyperconverged-group
  namespace: openshift-cnv
spec:
  targetNamespaces:
  - openshift-cnv
---
apiVersion: operators.coreos.com/v1alpha1
kind: Subscription
metadata:
  name: hco-operatorhub
  namespace: openshift-cnv
spec:
  source: redhat-operators
  sourceNamespace: openshift-marketplace
  name: kubevirt-hyperconverged
  channel: stable
  installPlanApproval: Automatic
EOF

ステップ2: オペレーターのインストールを確認

オペレーターのインストールを完了させ、そのステータスを確認します:

# CSVのステータスを`Succeeded`になるまで監視します
oc get csv -n openshift-cnv -w

インストールが完了した場合の期待される出力:

NAME                                       DISPLAY                    VERSION   REPLACES                                   PHASE
kubevirt-hyperconverged-operator.v4.20.3   OpenShift Virtualization   4.20.3    kubevirt-hyperconverged-operator.v4.20.1   Succeeded

`Succeeded`を見たら、`Ctrl+C`を使用して監視を停止します。

ステップ3: HyperConvergedリソースの作成

HyperConvergedカスタムリソースをデプロイしてインストールを完了させます:

oc apply -f - <<EOF
apiVersion: hco.kubevirt.io/v1beta1
kind: HyperConverged
metadata:
  name: kubevirt-hyperconverged
  namespace: openshift-cnv
spec:
EOF

注: ここで示されている最小限のHyperConverged仕様は、すべてのデフォルト設定を使用します。高度な構成オプションについては、公式ドキュメントを参照してください。

ステップ4: インストールの完了を待機

HyperConvergedリソースが利用可能になるまで待ちます:

oc wait hyperconverged kubevirt-hyperconverged -n openshift-cnv --for=condition=Available --timeout=15m

インストールは通常5-10分かかります。

ステップ5: インストールの確認

すべての条件が正常になるかどうかを確認します:

oc get hyperconverged kubevirt-hyperconverged -n openshift-cnv -o jsonpath='{range .status.conditions[*]}{.type}: {.status}{"\n"}{end}'

インストールが完了した場合の期待される出力:

ReconcileComplete: True
Available: True
Progressing: False
Degraded: False
Upgradeable: True

OpenShift仮想化のアンインストール

クラスターからOpenShift仮想化を削除する必要がある場合は、以下の順序で手順に従ってください。アンインストールプロセスは、すべての仮想化コンポーネントをグレースフルに終了するため、数分かかります。

重要: オペレーターのサブスクリプションのみを削除すると、デプロイされたリソースは削除されません。HyperConvergedリソースを最初に削除することで、すべてのコンポーネントを適切にクリーンアップする必要があります。

ステップ1: すべての仮想マシンの削除

アンインストール前に、すべてのVMが停止され、削除されていることを確認します:

# すべての名前空間でのVMのリスト表示
oc get vm -A

# 名前空間ごとにVMを削除します (各名前空間で)
oc delete vm --all -n <namespace>

警告: アンインストール中に実行中のVMは強制的に終了されます。

ステップ2: HyperConvergedリソースの削除

このステップは、すべてのOpenShift仮想化コンポーネントを削除します:

oc delete hyperconverged kubevirt-hyperconverged -n openshift-cnv

クリーンアップの進捗を監視します:

# ポッドが終了するのを5-10分待ちます (これは必要に応じて長くなる可能性があります)
oc get pods -n openshift-cnv -w

すべてのポッドが終了するまで待ちます。

ステップ3: オペレーターのサブスクリプションとCSVの削除

# サブスクリプションを削除
oc delete subscription hco-operatorhub -n openshift-cnv

# ClusterServiceVersionを削除
oc delete csv -n openshift-cnv -l operators.coreos.com/kubevirt-hyperconverged.openshift-cnv

ステップ4: 名前空間の削除 (オプション)

完全なクリーンアップのために名前空間を削除します:

oc delete namespace openshift-cnv

ステップ5: クリーンアップの確認

すべてのリソースが削除されていることを確認します:

# 空または"No resources found"を返すはずです
oc get pods -n openshift-cnv
oc get csv -n openshift-cnv

要約

OpenShift仮想化をインストールして構成しました。クラスターは、コンテナワークロードと仮想マシンを同時に実行できるようになりました。
