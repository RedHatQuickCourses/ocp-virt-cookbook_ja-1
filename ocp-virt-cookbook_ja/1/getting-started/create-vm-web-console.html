テンプレートを使用したWebコンソールからのVMの作成

概要

このチュートリアルでは、OpenShiftのWebコンソールと事前に構築されたテンプレートを使用して仮想マシンを作成する方法を示します。Webコンソールは、VM管理において直感的なグラフィカルインターフェースを提供し、新手ユーザーにとって理想的な始め方となります。

前提条件

  • OpenShift 4.18以上にOpenShift Virtualizationオペレーターがインストールされている（OpenShift Virtualizationのインストール方法を参照してください）

  • OpenShift Webコンソールへのアクセス権限を持ち、VMを作成できる

  • VMディスクのプロビジョニング用のデフォルトのStorageClassが構成されている

  • クラスター内に利用可能なブートソース画像（DataSources）がある

ブートソースが利用可能であることを確認します：

oc get datasources -n openshift-virtualization-os-images

ステップ1：仮想化セクションにアクセスする

  1. OpenShift Webコンソールにログインします

  2. 左側のナビゲーションメニューで、仮想化をクリックします

  3. VirtualMachinesをクリックしてVM管理インターフェースを表示します

VirtualMachinesページは、選択したプロジェクト内のすべてのVMとその状態、基本情報を表示します。

ステップ2：利用可能なテンプレートを探索する

OpenShift Virtualizationには、一般的なオペレーティングシステム用に事前に構成されたテンプレートが含まれています。

  1. VirtualMachinesページから、Create → From templateをクリックします

  2. オペレーティングシステム別にテンプレートを組織されたものを閲覧します：

    • Red Hat Enterprise Linux - さまざまなRHELバージョン

    • Fedora - 最新のFedoraリリース

    • CentOS Stream - CentOS Streamバージョン

    • Microsoft Windows - Windows Serverおよびデスクトップバージョン

Source availableを表示するテンプレートは、クラスター内にデフォルトのStorageClassが構成されている場合に、既に設定されたブートソースが使用可能です。

各テンプレートには次のものが含まれます：

  • CPUとメモリの事前構成

  • OSに適したディスクサイズ

  • ブートソース参照（DataSource）

  • 適用可能な場合のcloud-initまたはsysprep構成

ステップ3：テンプレートからVMを作成する

この例では、Fedora VMを作成します。他のオペレーティングシステムについても同様の手順が適用されます。

テンプレートの選択

  1. Fedoraテンプレートタイルをクリックします

  2. 右側のパネルでテンプレートの詳細を確認します

  3. Customize VirtualMachineをクリックします

VMの詳細の構成

Customize VirtualMachineページで、OverviewタブでVM設定を構成します：

  1. Name: VMの名前を入力します（例：fedora-test-vm）

  2. Project: VMが作成されるプロジェクト（namespace）を選択します

  3. Boot source: Fedora DataSourceがデフォルトで表示されるはずです

  4. CPU: CPUコアの数を調整します（デフォルトは通常1です）

  5. Memory: RAMの割り当てを調整します（各テンプレートによって異なります）

オプション：ストレージのカスタマイズ

ディスクを変更するために、Disksタブをクリックします：

  1. ルートディスクの構成を確認します

  2. 必要に応じてディスクサイズを調整するか、追加ディスクを追加します

  3. 必要に応じて、StorageClassが適切であることを確認します

確認して作成

すべてのタブで設定を確認します：

  1. Create VirtualMachineをクリックします

  2. VMの作成後にVMを自動的に起動するようにオプションでチェックします

ステップ4：VMの作成を監視する

Createをクリックすると、VMの詳細ページにリダイレクトされます。

プロビジョニングの進捗を追跡する

作成中にVMはいくつかの段階を経験します：

  1. プロビジョニング - ディスクはブートソースからクローンされます

  2. Starting - VMが起動します（起動が選択された場合）

  3. Running - VMは完全に動作します

Eventsタブを確認して、プロビジョニングの詳細な手順を確認します：

# CLIからのイベントの監視
oc get events -n <your-namespace> --field-selector involvedObject.name=<vm-name> -w

VMが実行中であることを確認する

プロビジョニングが完了すると、ステータスがRunningに変更されます。

ステップ5：VMコンソールにアクセスする

Webコンソールは、VMのグラフィカルコンソールに直接アクセスします。

コンソールを開く

  1. VMの詳細ページで、Consoleタブをクリックします

  2. ブラウザでVNCコンソールが直接ロードされます

コンソールオプション

コンソールバーには、以下のオプションが提供されます：

  • Send key - 特殊キー組み合わせ（Ctrl+Alt+Delなど）を送信します

  • Disconnect - コンソール接続を終了します

  • Guest credentials - テンプレートが提供する場合のユーザー名とパスワード

性能を向上させるために、Linux VMへのSSHアクセスまたはWindows VMへのRDPアクセスをWebコンソールよりも定期的に使用することを検討してください。

ゲストログイン

Fedoraや他のクラウドイメージの場合、cloud-initの構成に依存するデフォルトのユーザー名とパスワードがあります：

  • cloud-initユーザーデータが提供されなかった場合は、ユーザーを設定する必要がある場合があります

  • テンプレートのドキュメントを参照してデフォルトのユーザー名とパスワードを確認してください

  • テンプレート内のcloud-initをカスタマイズして自動ユーザー設定を行うことを検討してください

ステップ6：基本的なVM操作

Webコンソールは、一般的なVM操作に簡単にアクセスします。

アクションメニュー

VMの詳細ページまたはVirtualMachinesリストから、アクションメニューをアクセスします：

VMを停止する

  1. Actions → Stopをクリックします

  2. 必要に応じてアクションを確認します

  3. VMはグレースフルにシャットダウンします

# CLIの等価
virtctl stop <vm-name>

VMを再起動する

  1. Actions → Restartをクリックします

  2. VMはグレースフルに再起動します

# CLIの等価
virtctl restart <vm-name>

パスとアンパス

VMをパスするには、その状態をフリーズするだけで、シャットダウンしません：

  1. Actions → Pauseをクリックします

  2. VMの状態はメモリーに保存されます

  3. Actions → Unpauseをクリックして再開します

パスされたVMは、ノード上のメモリーリソースを消費します。

VMを削除する

  1. Actions → Deleteをクリックします

  2. 関連ディスク（PVC）を削除するかどうかを選択します

  3. 削除を確認します

確認ダイアログでDelete diskオプションをチェックしないでください。これにより、VMディスク（DataVolume）を削除後にもデータボリュームを再利用して新しいVMを作成できます。
ディスクとVMを削除すると、永久に削除されます。必要に応じてバックアップを確保してください。

ステップ7：VMの詳細とメトリックの表示

VMの詳細ページは、モニタリング情報と構成情報を提供します。

概要タブ

重要な情報を一目で確認できます：

  • VMの状態と条件

  • ゲストエージェントがインストールされている場合のIPアドレス

  • CPUとメモリーの利用率

  • ブート順序とデバイス

メトリックタブ

リアルタイムのパフォーマンスメトリックを表示します：

  • CPUの利用率の時間の経過とともに

  • メモリーの利用率

  • ネットワークスループット

  • ストレージI/O

完全なメトリックは、QEMUゲストエージェントがVMにインストールされている必要があります。

YAMLタブ

VMの定義を直接表示および編集できます：

  1. YAMLタブをクリックします

  2. VirtualMachineマニフェストを確認または編集します

  3. Saveをクリックして変更を適用します

YAMLファイルをダウンロードするには、Downloadボタンをクリックします。

クイック作成を使用してVMを作成する

デフォルトの設定でより速くVMを作成するには、クイック作成オプションを使用します。

  1. VirtualMachinesページから、Create → From templateをクリックします

  2. テンプレートを選択します

  3. Customize VirtualMachineではなく、Quick create VirtualMachineをクリックします

  4. デフォルトの設定でVMが即座に作成されます

これは、以下のような場合に役立ちます：

  • テストおよび開発環境

  • デフォルトのテンプレート設定が十分である場合

  • 迅速なプロトタイピング

要約
