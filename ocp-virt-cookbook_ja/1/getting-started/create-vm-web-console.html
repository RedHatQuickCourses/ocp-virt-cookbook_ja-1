ウェブコンソールとテンプレートを使用したVMの作成

概要

このチュートリアルでは、OpenShiftウェブコンソールと事前構築済みテンプレートを使用して仮想マシンを作成する方法を説明します。ウェブコンソールはVM管理のための直感的なグラフィカルインターフェースを提供し、新規ユーザーにとって理想的な出発点となります。

前提条件

  • OpenShift VirtualizationオペレーターがインストールされたOpenShift 4.18+ (参照: OpenShift Virtualizationのインストール)

  • VMを作成する権限を持つOpenShiftウェブコンソールへのアクセス

  • VMディスクプロビジョニング用に構成されたデフォルトのStorageClass

  • クラスター内で利用可能なブートソースイメージ (DataSource)

ブートソースが利用可能であることを確認します。

oc get datasources -n openshift-virtualization-os-images

ステップ 1: 仮想化セクションへのアクセス

  1. OpenShiftウェブコンソールにログインします

  2. 左側のナビゲーションメニューで、Virtualization をクリックします

  3. VirtualMachines をクリックして、VM管理インターフェースを表示します

VirtualMachinesページには、選択したプロジェクト内のすべてのVMが、そのステータスと基本情報とともに表示されます。

ステップ 2: 利用可能なテンプレートの参照

OpenShift Virtualizationには、一般的なオペレーティングシステム用の事前設定済みテンプレートが含まれています。

  1. VirtualMachinesページから、Create → From template をクリックします

  2. オペレーティングシステム別に整理された利用可能なテンプレートを参照します。

    • Red Hat Enterprise Linux - さまざまなRHELバージョン

    • Fedora - 最新のFedoraリリース

    • CentOS Stream - CentOS Streamバージョン

    • Microsoft Windows - Windows Serverおよびデスクトップバージョン

Source available と表示されているテンプレートには、すぐに使用できる事前設定済みのブートソースがあります。これには、クラスターでdefault StorageClass が構成されている必要があります。

各テンプレートには以下が含まれます。

  • 事前設定されたCPUおよびメモリー設定

  • OSに適したディスクサイズ

  • ブートソース参照 (DataSource)

  • Cloud-initまたはsysprep構成 (該当する場合)

ステップ 3: テンプレートからのVM作成

この例ではFedora VMを作成します。他のオペレーティングシステムでも同様のプロセスです。

テンプレートの選択

  1. Fedora テンプレートタイルをクリックします

  2. サイドパネルに表示されるテンプレートの詳細を確認します

  3. Customize VirtualMachine をクリックします

VM詳細の構成

Customize VirtualMachine ページで、Overview タブのVM設定を構成します。

  1. 名前: VMの名前を入力します (例: fedora-test-vm)

  2. プロジェクト: VMが作成されるプロジェクト (ネームスペース) を選択します

  3. ブートソース: デフォルトでFedora DataSourceが表示されるはずです

  4. CPU: CPUコア数を調整します (デフォルトは通常1です)

  5. メモリー: RAM割り当てを調整します (デフォルトはテンプレートによって異なります)

ストレージのカスタマイズ (オプション)

ストレージを変更するには、Disks タブをクリックします。

  1. ルートディスク構成を確認します

  2. オプションでディスクサイズを調整したり、追加ディスクを追加したりします

  3. StorageClassがニーズに適していることを確認します

確認と作成

  1. すべてのタブの設定を確認します

  2. Create VirtualMachine をクリックします

  3. オプションで、Start this VirtualMachine after creation をチェックしてすぐに起動します

ステップ 4: VM作成の監視

Createをクリックすると、VM詳細ページにリダイレクトされます。

プロビジョニングの進行状況を追跡する

VMは作成中にいくつかのフェーズを経ます。

  1. Provisioning - ディスクがブートソースからクローンされています

  2. Starting - VMが起動しています (起動が選択されていた場合)

  3. Running - VMが完全に動作しています

Events タブを監視して、詳細なプロビジョニングステップを確認します。

# Alternatively, watch events from CLI
oc get events -n <your-namespace> --field-selector involvedObject.name=<vm-name> -w

VMが実行中であることを確認する

プロビジョニングが完了すると、ステータスが Running に変わります。

ステップ 5: VMコンソールへのアクセス

ウェブコンソールは、VMのグラフィカルコンソールへの直接アクセスを提供します。

コンソールを開く

  1. VM詳細ページで、Console タブをクリックします

  2. VNCコンソールがブラウザに直接ロードされます

コンソールオプション

コンソールツールバーには以下のオプションがあります。

  • Send key - 特殊なキーの組み合わせを送信します (Ctrl+Alt+Delなど)

  • Disconnect - コンソール接続を閉じます

  • Guest credentials - テンプレートが提供する場合のユーザー名とパスワード

パフォーマンスを向上させるには、通常の作業ではウェブコンソールの代わりにLinux VMへのSSHアクセスまたはWindows VMへのRDPを使用することを検討してください。

ゲストログイン

Fedoraおよびその他のクラウドイメージの場合、デフォルトの資格情報はcloud-init構成に依存します。

  • cloud-initユーザーデータが提供されていない場合、ユーザーを設定する必要があるかもしれません

  • デフォルトの資格情報については、テンプレートのドキュメントを確認してください

  • 自動化されたユーザー設定のために、テンプレート内でcloud-initをカスタマイズすることを検討してください

ステップ 6: 基本的なVM操作

ウェブコンソールは、一般的なVM操作に簡単にアクセスできるようにします。

VMアクションメニュー

VM詳細ページまたはVirtualMachinesリストから、アクションメニューにアクセスします。

VMを停止する

  1. Actions → Stop をクリックします

  2. プロンプトが表示されたらアクションを確認します

  3. VMは正常にシャットダウンします

# CLI equivalent
virtctl stop <vm-name>

VMを再起動する

  1. Actions → Restart をクリックします

  2. VMは正常な再起動を実行します

# CLI equivalent
virtctl restart <vm-name>

一時停止と再開

VMを一時停止すると、シャットダウンせずにその状態がフリーズします。

  1. Actions → Pause をクリックします

  2. VMの状態はメモリに保持されます

  3. Actions → Unpause をクリックして再開します

一時停止されたVMは、ノード上でメモリリソースを消費し続けます。

VMを削除する

  1. Actions → Delete をクリックします

  2. 関連付けられたディスク (PVC) を削除するかどうかを選択します

  3. 削除を確認します

削除後にVMディスク (DataVolume) を保持するには、確認ダイアログに表示される Delete disk オプションのチェックを外します。これにより、後で新しいVMでディスクを再利用できます。
ディスクとともにVMを削除すると、元に戻せません。必要に応じてバックアップがあることを確認してください。

ステップ 7: VMの詳細とメトリクスの表示

VM詳細ページには、監視情報と構成情報が表示されます。

概要タブ

主要な情報が一目でわかります。

  • VMステータスと条件

  • IPアドレス (ゲストエージェントがインストールされている場合)

  • CPUおよびメモリー使用率

  • ブートオーダーとデバイス

メトリクスタブ

リアルタイムのパフォーマンスメトリクスを表示します。

  • 経時的なCPU使用率

  • メモリー使用率

  • ネットワークスループット

  • ストレージI/O

完全なメトリクスには、VMにQEMUゲストエージェントがインストールされている必要があります。

YAMLタブ

VM定義を直接表示および編集します。

  1. YAML タブをクリックします

  2. VirtualMachineマニフェストを確認または変更します

  3. Save をクリックして変更を適用します

Download ボタンをクリックしてYAMLファイルをダウンロードすることもできます。

クイック作成を使用したVMの作成

デフォルト設定でVMをより速く作成するには、クイック作成オプションを使用します。

  1. VirtualMachinesページから、Create → From template をクリックします

  2. テンプレートを選択します

  3. Customize VirtualMachine の代わりに、Quick create VirtualMachine をクリックします

  4. VMはデフォルト設定ですぐに作成されます

これは以下の場合に役立ちます。

  • テストおよび開発環境

  • デフォルトのテンプレート設定で許容できる場合

  • 迅速なプロトタイピング

まとめ

あなたは次の方法を学びました。

  • ウェブコンソールの仮想化セクションに移動する

  • VMテンプレートを参照および選択する

  • カスタマイズされた設定でVMを作成する

  • VMプロビジョニングの進行状況を監視する

  • ウェブブラウザを介してVMコンソールにアクセスする

  • 基本的なVM操作 (停止、起動、再起動、削除) を実行する

  • VMの詳細とメトリクスを表示する
