前提条件とセットアップ

概要

このページでは、本書に記載されている OpenShift Virtualization を使用するための前提条件と要件を概説します。

クラスター要件

OpenShift のバージョン

  • OpenShift 4.18+ 以降

  • OpenShift Virtualization オペレーターがインストールされ、実行中であること

クラスターアクセス

実行したいタスクに応じて、適切なアクセスレベルが必要です。

  • Cluster Admin: オペレーターのインストール、ストレージ構成、ネットワーク構成に必要です

  • Namespace Admin: 特定のネームスペース内で VM を管理するのに十分です

  • Developer: 適切な RBAC 権限があれば、VM の作成と管理が可能です

必須の CLI ツール

OpenShift CLI (oc)

OpenShift コマンドラインインターフェースは、クラスター操作に不可欠です。

インストール元: OpenShift CLI ドキュメント

インストールの確認:

oc version

Kubernetes CLI (kubectl)

oc が推奨されますが、kubectl も多くの操作に使用できます。

インストール元: Kubernetes ドキュメント

インストールの確認:

kubectl version

Virtctl

virtctl ツールは、KubeVirt/OpenShift Virtualization リソースを管理するために特別に設計されています。

インストール元: OpenShift Virtualization ドキュメント

インストールの確認:

virtctl version

オプションのツール

ワークフローによっては、以下のツールも必要となる場合があります。

  • kustomize: Kubernetes 構成を管理するため

  • helm: パッケージ化されたアプリケーションをデプロイするため

  • ansible: 自動化とオーケストレーションのため

クラスター構成

ストレージ

クラスターに以下のものがあることを確認してください。

  • 少なくとも 1 つの StorageClass が構成されている

  • VM ディスクに十分なストレージ容量がある

  • 動的プロビジョニングをサポートするストレージプロビジョナーがある

利用可能なストレージクラスを確認する:

oc get storageclass

ネットワーク

ネットワーク接続を確認する:

  • クラスターネットワークが機能している

  • Pod ネットワークにアクセスできる

  • VM に外部ネットワークアクセスが必要な場合は、それが可能である

確認

チュートリアルに進む前に、セットアップを確認してください。

# Check cluster status
oc get nodes

# Verify OpenShift Virtualization operator
oc get csv -n openshift-cnv

# Check available VM templates
oc get datasources -n openshift-virtualization-os-images

次のステップ

前提条件が整ったら、次へ進みます。

  • virtctl の基本を学ぶ

  • デフォルトストレージを構成する
