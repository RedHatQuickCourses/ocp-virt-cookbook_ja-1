はじめに

概要

このモジュールでは、OpenShift Virtualization を使い始めるために必要な必須ツールと基本的な設定について説明します。OpenShift Virtualization を初めて使用する場合でも、知識を再確認したい場合でも、これらのガイドはすぐに作業を開始するのに役立ちます。

学習内容

このセクションでは、以下について学習します。

  • OpenShift Virtualization を操作するための前提条件と必要なツール

  • 仮想マシンを管理するための virtctl コマンドラインツールの基本的な使用方法

  • 仮想マシンデプロイメントのデフォルトストレージを設定する方法

  • Cloud-init の基礎と VM 初期化パターン

前提条件

このセクションのチュートリアルに進む前に、以下を準備してください。

  • OpenShift Virtualization オペレーターがインストールされた OpenShift 4.18 以降のクラスター

  • クラスター管理者アクセス、または適切な RBAC 権限

  • CLI ツールがインストールされていること: oc、kubectl、virtctl

  • Kubernetes/OpenShift の概念に関する基本的な理解

セクション

前提条件とセットアップ

OpenShift Virtualization を操作するための詳細な要件とセットアップ手順。

OpenShift Virtualization Operator のインストールと設定

クラスターに OpenShift Virtualization オペレーターをインストールして設定します。

VM管理のためのvirtctlの使用

仮想マシンを管理するための必須の virtctl コマンドを学習します。

デフォルトストレージクラスの設定

VM デプロイメントを簡素化するためにデフォルトのストレージクラスを設定します。

ウェブコンソールとテンプレートを使用したVMの作成

テンプレートを使用して OpenShift Web コンソールから仮想マシンを作成します。

VMライフサイクルステートと基本操作の理解

効果的な VM 管理のための VM ライフサイクル状態と基本的な操作を理解します。

OpenShift Virtualization VMのためのCloud-initの基本

cloud-init の基礎、実行ステージ、一般的なモジュール、および VirtualMachine マニフェストに cloud-init 設定を埋め込む方法を学習します。

次のステップ

このモジュールを完了すると、以下を探索する準備が整います。

  • ストレージ設定 - ストレージオプションの詳細

  • ネットワーク設定 - VM のネットワークセットアップ

  • Cloud-init のトラブルシューティング - cloud-init の問題に対する診断手順
