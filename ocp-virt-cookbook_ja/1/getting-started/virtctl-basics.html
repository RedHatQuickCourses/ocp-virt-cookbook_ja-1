virtctl を使用した VM 管理

概要

virtctl CLI ツールは、OpenShift Virtualization で仮想マシンを管理するためのコマンドを提供します。このガイドでは、VM の主要な操作について説明します。

テスト済みのバージョン:

OpenShift 4.20

前提条件

  • OpenShift Virtualization がインストールされた OpenShift 4.18 以降

  • virtctl CLI ツールがインストールされていること

virtctl のダウンロードとインストール

virtctl CLI ツールは、OpenShift Web コンソールから直接ダウンロードできます。

  1. OpenShift Web コンソールで、左側のサイドバーから Virtualization → Overview に移動します。

    Figure 1. OpenShift コンソールの Virtualization Overview

  2. Overview ページの右上にある Download the virtctl command-line utility リンクをクリックします。

    Figure 2. virtctl ダウンロードリンク

  3. ダウンロードページで、ご使用のオペレーティングシステムとアーキテクチャーに適した virtctl バイナリーを選択します。

    Figure 3. 利用可能な virtctl バイナリー

  4. ダウンロード後、バイナリーを実行可能にし、PATH に移動します。

    # Linux/Mac
chmod +x virtctl
sudo mv virtctl /usr/local/bin/

# Verify installation
virtctl version

    Windows の場合は、実行ファイルを抽出し、その場所をシステムの PATH に追加します。

主要なコマンド

VM の一覧表示

VM を管理する前に、oc コマンドを使用してクラスターで利用可能な VM を一覧表示します。

# List all VMs in a specific namespace
oc get vm -n myapp

# List all VMs across all namespaces
oc get vm -A

出力例:

NAMESPACE   NAME            AGE   STATUS    READY
myapp       fedora-app-vm   13d   Running   True

実行中の VM インスタンス (VMI) を、IP アドレスやノード配置などの追加の詳細とともに表示するには、以下を実行します。

oc get vmi -A

出力例:

NAMESPACE   NAME            AGE   PHASE     IP             NODENAME   READY
myapp       fedora-app-vm   13d   Running   10.128.1.225   ocplab01   True
VM の一覧表示とクエリーには oc を使用します。起動、停止、コンソールアクセスなどの VM 操作には virtctl を使用します。

VM ライフサイクル

VM の起動、停止、再起動を行います。

# Start a VM
virtctl start fedora-app-vm -n myapp

# Stop a VM
virtctl stop fedora-app-vm -n myapp

# Restart a VM
virtctl restart fedora-app-vm -n myapp

VM へのアクセス

VM に接続します。

# Console access (press Ctrl+] to exit)
virtctl console fedora-app-vm -n myapp

# SSH access (requires SSH configured)
virtctl ssh user@fedora-app-vm -n myapp

# VNC access
virtctl vnc fedora-app-vm -n myapp

VM 情報

VM の詳細を取得します (VM に QEMU ゲストエージェントがインストールされている必要があります)。

# Guest OS information
virtctl guestosinfo fedora-app-vm -n myapp

出力例:

ファイル転送

virtctl scp を使用して、VM との間でファイルをコピーします。これには、VM で SSH が実行されていること、および次のいずれかの認証方法が必要です。

  • SSH キー (推奨): 公開キーは VM の ~/.ssh/authorized_keys に存在する必要があります

  • パスワード: VM で SSH のパスワード認証が有効になっている必要があります

# Copy to VM (uses default SSH key)
virtctl scp local-file.txt fedora@fedora-app-vm:/tmp/remote.txt -n myapp

# Copy from VM
virtctl scp fedora@fedora-app-vm:/tmp/remote.txt ./local.txt -n myapp

# Specify a different SSH key
virtctl scp -i ~/.ssh/my_key local-file.txt fedora@fedora-app-vm:/tmp/ -n myapp
SSH が設定されていない場合は、まず virtctl console を使用して VM にアクセスし、SSH キーを設定します。

VM の作成時に SSH キーを挿入するには、cloud-init 設定に含めます。

#cloud-config
user: fedora
ssh_authorized_keys:
  - ssh-rsa AAAAB3NzaC... your-key-here

SSH 設定オプションの詳細については、vm-configuration:remote-access-linux-vms.adoc を参照してください。

VM 操作の検証

VM を起動または停止した後、操作を検証します。

# Check VM status
oc get vm fedora-app-vm -n myapp

正常に起動した後の出力例:

NAME            AGE    STATUS    READY
fedora-app-vm   7d3h   Running   True
# Check VMI (running instance)
oc get vmi fedora-app-vm -n myapp

期待される結果: 実行中の場合は VMI がリストされ、停止後は VMI はリストされません。

クイックリファレンス

各アクセス方法を使用するタイミング:

  • virtctl console - テキストのみのターミナルアクセス。ネットワークなしで動作します。

  • virtctl ssh - キーが設定されている場合に便利な SSH

  • virtctl vnc - GUI ベースの VM 用のグラフィカルデスクトップアクセス

VM 情報を確認するタイミング:

  • virtctl guestosinfo - 更新前に OS バージョンを確認

  • virtctl fslist - ソフトウェアをインストールする前にディスク容量を確認

  • virtctl userlist - 現在ログインしているユーザーを確認

一般的な問題

問題: サーバーに接続できない

解決策: クラスターアクセスを確認します。

oc whoami
oc get nodes

問題: ゲストエージェントコマンドがデータを返さない

解決策: VM に QEMU ゲストエージェントをインストールします。インストール手順については、QEMU ゲストエージェントのインストール を参照してください。

問題: SSH/SCP コマンドが失敗する

解決策: VM で SSH が設定されていることを確認します。コンソールを使用して確認します。

# Access via console first
virtctl console fedora-app-vm -n myapp

# Inside VM, check SSH status
systemctl status sshd

問題: コンソールがハングする、または応答しない

解決策: Ctrl+] で終了し、VM ステータスを確認します。

oc get vmi fedora-app-vm -n myapp

ベストプラクティス

  • 混乱を避けるために、-n で必ず namespace を指定します。

  • 初期設定にはコンソールを使用し、その後は通常のアクセスには SSH を使用します。

  • すべての VM にゲストエージェントをインストールして、すべての機能を利用できるようにします。

  • VM のステータスには oc を、VM の操作には virtctl を使用します。

  • セッションがハングするのを防ぐために、Ctrl+] でコンソールを適切に終了します。

  • 外部アクセスを設定する前に、virtctl ssh でテストします。

まとめ

このチュートリアルでは、以下のことを学びました。

  • OpenShift Web コンソールから virtctl CLI ツールをダウンロード、インストール、設定する方法

  • 起動、停止、再起動を含む VM ライフサイクル操作を管理する方法

  • コンソール、SSH、および VNC の方法を使用して VM にアクセスする方法

  • QEMU ゲストエージェントを介して、ゲスト OS 情報、ファイルシステムの詳細、およびユーザーリストを取得する方法

  • virtctl scp を使用して、VM との間でファイルを転送する方法
