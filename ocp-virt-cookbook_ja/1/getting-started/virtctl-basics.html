VM管理のためのvirtctlの使用

概要

virtctl CLIツールは、OpenShift Virtualizationで仮想マシンを管理するためのコマンドを提供します。このガイドでは、基本的なVM操作について説明します。

テスト済みのバージョン：

OpenShift 4.20

前提条件

  • OpenShift VirtualizationがインストールされたOpenShift 4.18以降

  • virtctl CLIツールがインストールされていること

virtctlのダウンロードとインストール

virtctl CLIツールは、OpenShiftウェブコンソールから直接ダウンロードできます。

  1. OpenShiftウェブコンソールで、左側のサイドバーから 仮想化 → 概要 に移動します。

    Figure 1. OpenShiftコンソールにおける仮想化の概要

  2. 概要ページの右上にある virtctlコマンドラインユーティリティをダウンロード のリンクをクリックします。

    Figure 2. virtctlダウンロードリンク

  3. ダウンロードページで、お使いのオペレーティングシステムとアーキテクチャに適した virtctl バイナリを選択します。

    Figure 3. 利用可能なvirtctlバイナリ

  4. ダウンロード後、バイナリを実行可能にし、`PATH`に移動します。

    # Linux/Mac
chmod +x virtctl
sudo mv virtctl /usr/local/bin/

# Verify installation
virtctl version

    Windowsの場合、実行可能ファイルを解凍し、その場所をシステムの`PATH`に追加します。

必須コマンド

VMのリスト表示

VMを管理する前に、`oc`コマンドを使用してクラスター内で利用可能なVMをリスト表示します。

# List all VMs in a specific namespace
oc get vm -n myapp

# List all VMs across all namespaces
oc get vm -A

出力例：

NAMESPACE   NAME            AGE   STATUS    READY
myapp       fedora-app-vm   13d   Running   True

IPアドレスやノード配置などの詳細情報とともに、実行中のVMインスタンス（VMI）を表示するには：

oc get vmi -A

出力例：

NAMESPACE   NAME            AGE   PHASE     IP             NODENAME   READY
myapp       fedora-app-vm   13d   Running   10.128.1.225   ocplab01   True
VMのリスト表示とクエリには oc を使用します。起動、停止、コンソールアクセスなどのVM操作には virtctl を使用します。

VMライフサイクル

VMの起動、停止、再起動：

# Start a VM
virtctl start fedora-app-vm -n myapp

# Stop a VM
virtctl stop fedora-app-vm -n myapp

# Restart a VM
virtctl restart fedora-app-vm -n myapp

VMへのアクセス

VMに接続します。

# Console access (press Ctrl+] to exit)
virtctl console fedora-app-vm -n myapp

# SSH access (requires SSH configured)
virtctl ssh user@fedora-app-vm -n myapp

# VNC access
virtctl vnc fedora-app-vm -n myapp

VM情報

VMの詳細情報を取得します（VMにQEMUゲストエージェントがインストールされている必要があります）：

# Guest OS information
virtctl guestosinfo fedora-app-vm -n myapp

ファイル転送

virtctl scp を使用して、VMとの間でファイルをコピーします。これには、VMでSSHが実行されており、以下の認証方法のいずれかが必要です。

  • SSHキー (推奨): 公開キーはVMの ~/.ssh/authorized_keys に存在する必要があります。

  • パスワード: VMでSSHのパスワード認証が有効になっている必要があります。

# Copy to VM (uses default SSH key)
virtctl scp local-file.txt fedora@fedora-app-vm:/tmp/remote.txt -n myapp

# Copy from VM
virtctl scp fedora@fedora-app-vm:/tmp/remote.txt ./local.txt -n myapp

# Specify a different SSH key
virtctl scp -i ~/.ssh/my_key local-file.txt fedora@fedora-app-vm:/tmp/ -n myapp
SSHが設定されていない場合、最初に virtctl console を使用してVMにアクセスし、SSHキーを設定します。

VM作成時にSSHキーを挿入するには、cloud-init設定に含めます。

#cloud-config
user: fedora
ssh_authorized_keys:
  - ssh-rsa AAAAB3NzaC... your-key-here

詳細なSSH設定オプションについては、vm-configuration:remote-access-linux-vms.adoc を参照してください。

VM操作の検証

VMの起動または停止後、操作を検証します。

# Check VM status
oc get vm fedora-app-vm -n myapp

正常に起動した後の出力例：

NAME            AGE    STATUS    READY
fedora-app-vm   7d3h   Running   True
# Check VMI (running instance)
oc get vmi fedora-app-vm -n myapp

予想: 実行中はVMIがリスト表示され、停止後はVMIが表示されません。

クイックリファレンス

各アクセス方法の使用時期：

  • virtctl console - テキスト専用のターミナルアクセス。ネットワークなしで動作します。

  • virtctl ssh - キーが設定されている場合に便利なSSHアクセス。

  • virtctl vnc - GUIベースのVM向けのグラフィカルデスクトップアクセス。

VM情報の確認時期：

  • virtctl guestosinfo - 更新前にOSのバージョンを確認します。

  • virtctl fslist - ソフトウェアをインストールする前にディスク容量を確認します。

  • virtctl userlist - 現在ログインしているユーザーを確認します。

よくある問題

問題: サーバーに接続できない

解決策: クラスターへのアクセスを確認します。

oc whoami
oc get nodes

問題: ゲストエージェントコマンドがデータを返さない

解決策: VMにQEMUゲストエージェントをインストールします。インストール手順については、QEMUゲストエージェントのインストール を参照してください。

問題: SSH/SCPコマンドが失敗する

解決策: VMでSSHが設定されていることを確認します。コンソールを使用して確認します。

# Access via console first
virtctl console fedora-app-vm -n myapp

# Inside VM, check SSH status
systemctl status sshd

問題: コンソールがハングしたり、応答しない

解決策: Ctrl+] で終了し、VMのステータスを確認します。

oc get vmi fedora-app-vm -n myapp

ベストプラクティス

  • 混乱を避けるために、-n を使用して常に名前空間を指定します。

  • 初期設定にはコンソールを使用し、その後は通常のアクセスにはSSHを使用します。

  • 完全な機能のために、すべてのVMにゲストエージェントをインストールします。

  • VMのステータス確認には oc を使用し、VM操作には virtctl を使用します。

  • セッションがハングするのを避けるため、Ctrl+] でコンソールを適切に終了します。

  • 外部アクセスを設定する前に、virtctl ssh でテストします。

  • コンソール、SSH、およびVNCメソッドを使用してVMにアクセスする方法

  • QEMUゲストエージェントを介して、ゲストOS情報、ファイルシステムの詳細、およびユーザーリストを取得する方法

  • virtctl scp を使用してVMとの間でファイルを転送する方法
